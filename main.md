# sound escape 要件定義書

**版:** 1.0  
**作成日:** 2026-06-21  
**対象:** PCブラウザ向けAIハッカソン作品  
**実装方針:** コア体験を3時間で完成させ、物理精度とコンテンツ量は段階的に拡張する

---

## 1. プロダクト概要

プレイヤーは、夜間専門の忘れ物回収業者である。  
人が立ち入れない深夜の屋敷や閉店後のホームセンターへ遠隔操作ロボットを投入し、依頼された忘れ物を探して回収する。

現場は停電しており、通常の照明を使うと幽霊に位置を知られてしまう。プレイヤーはマイクに向かって声を出し、その音から生じる円形の波と反響をレーダーとして利用しなければならない。

音を出せば周囲が見えるが、幽霊にも発見される。  
プレイヤーは声量、移動、探索頻度を調整しながら、忘れ物を見つけて入口まで持ち帰る。

> **声を出さなければ進めない。声を出すほど、何かが近づいてくる。**

---

## 2. 目的

### 2.1 プロダクト目標

- 「波」というテーマを、見た目ではなくゲームルールとして体験させる。
- 声を、移動・探索・危険のすべてに関係する主入力にする。
- 暗闇の中で音波により空間を読む、新しい探索体験を提供する。
- URLを開いて45秒以内にコア体験へ到達できるようにする。
- APIキー、DB、画像素材、音声素材がなくても1プレイ完結させる。

### 2.2 成功条件

プレイヤーが初回プレイ中に以下を理解できること。

1. 声を出すとロボットが動く。
2. 声を出すと円状の音波が広がる。
3. 波が壁や物体へ当たると、その場所が一時的に見える。
4. 波は壁で遮断・反射・減衰する。
5. 大きな音ほど遠くまで見えるが、幽霊に発見されやすい。
6. 忘れ物を回収して入口へ戻ればクリアになる。

## 5. 世界観

### 5.1 主人公

主人公は「夜間遺失物回収局」のオペレーターであり、現場へ直接入らず、忘れ物回収ロボットを遠隔操作する。

### 5.2 舞台

MVPでは**閉店後のホームセンター**を採用する。

理由:

- 棚、資材、金属、布、木材など音の反射が異なる物体を自然に配置できる。
- 長い通路と分岐が見下ろし型ゲームに適している。
- 忘れ物の種類を作りやすい。
- 屋敷よりもロボットを投入する理由を説明しやすい。

拡張版では以下を追加できる。

- 古い屋敷
- 学校
- 病院
- ホテル
- 工場
- 地下駐車場

### 5.3 夜間限定の理由

- 昼間は店舗や施設が営業中であり、ロボットを投入できない。
- 深夜にのみ異常現象が発生し、通常の人間は立ち入れない。
- 照明設備は幽霊の干渉により使用できない。
- 非常灯を点けると幽霊へ即座に位置を知られる。

## 7. プレイヤー操作要件

### 7.1 移動

| 操作 | 機能 |
|---|---|
| W / ↑ | 上へ移動 |
| S / ↓ | 下へ移動 |
| A / ← | 左へ移動 |
| D / → | 右へ移動 |
| E | 忘れ物・設備を調べる |
| Shift | 短時間ブーストまたは緊急後退 |
| Space | 手動Ping |
| Esc | 一時停止 |

### 7.2 発声必須ルール

- ロボットは発声中、または直前の発声エネルギーが残っている間のみ移動できる。
- 無音状態が一定時間続くと徐々に減速し、停止する。
- 方向入力だけではステージを最後まで攻略できない。
- マイク利用拒否時のみ、手動Pingと疑似動力モードを利用できる。
- 手動モードはデモとアクセシビリティ用であり、通常モードよりスコア倍率を低くする。

### 7.3 声量と効果

| 声量 | 移動速度 | 探索距離 | 幽霊への露見 |
|---|---:|---:|---:|
| 小声 | 低 | 近距離 | 低 |
| 普通 | 標準 | 中距離 | 中 |
| 大声 | 高 | 長距離 | 高 |
| 無音 | 停止 | なし | 低下 |

## 8. 音響レーダー要件

### 8.1 表示コンセプト

- 画面は黒に近い暗色を基調とする。
- 音波はレーダー画面のようなネオングリーンで表示する。
- プレイヤーを中心に複数の円形波面が広がる。
- 波が当たった壁、棚、床、物体の輪郭を一時的に発光させる。
- 当たっていない方向の波だけが先へ進む。
- 障害物に当たった部分は欠け、残った波面がさらに外側へ伝播する。
- 反射や通過を繰り返すたびに波の明度、太さ、到達距離を減衰させる。
- 過去に確認した場所は短時間だけ薄い残像として残す。

### 8.2 視覚表現

| 対象 | 表示 |
|---|---|
| 壁・棚 | 緑色の輪郭 |
| 通路・床 | 暗い緑の面表示 |
| 忘れ物 | 黄色または白緑の二重反響 |
| 幽霊 | 赤紫のノイズ状シルエット |
| 危険床 | 赤紫の脈動 |
| 出入口 | 一定周期で点滅するマーカー |
| ロボット | 中央に明るい円形マーカー |

### 8.3 表示時間

- 波面表示: 0.6〜1.2秒
- 地形輪郭: 1.5〜4秒
- 幽霊シルエット: 0.2〜0.6秒
- 危険床: 一度発見後、2〜5秒
- 古い情報ほど暗くする

### 8.4 音量検出

Web Audio APIでRMS音量を測定する。

推奨初期値:

- ノイズフロア: 起動時1〜2秒間で自動測定
- 小Ping: ノイズフロア + 0.03以上
- 標準Ping: RMS 0.08以上が150〜250ms継続
- 大Ping: RMS 0.20以上
- Pingクールダウン: 500〜800ms
- 感度は設定画面で変更可能

## 9. 音波計算方式

本要件では、**ハッカソンMVP**と**完成版**で方式を分ける。

### 9.1 MVP方式: 物理近似型レイ・ウェーブハイブリッド

3時間開発では、完全な波動方程式ではなく、360度方向へ多数の音線を発射して波面を構成する。

#### 処理

1. ロボット位置から360度へ128〜360本のレイを発射する。
2. 各レイを一定速度で前進させる。
3. 壁へ衝突したレイは法線を使って反射方向を計算する。
4. 反射ごとにエネルギーを減衰させる。
5. エネルギーがしきい値未満になったレイを削除する。
6. 同じ反射回数のレイ先端を線でつなぎ、欠けた円形波として描画する。
7. 反射は最大1〜2回とする。
8. 狭い開口部では、通過したレイの角度を少し拡散させ、簡易回折を表現する。

#### 反射計算

入射方向を `d`、壁の法線を `n` とする。

```text
r = d - 2(d・n)n
```

#### 減衰

```text
energyNext =
  energyCurrent
  × distanceAttenuation
  × materialReflection
  × bounceAttenuation
```

推奨初期値:

- 距離減衰: 0.985〜0.995 / 更新
- 反射減衰: 0.55〜0.85
- 最大反射回数: 2
- 最小描画エネルギー: 0.05

### 9.2 完成版方式: 2D FDTD波動シミュレーション

完成版では、空間を格子化し、2次元波動方程式の有限差分時間領域法で音圧場を更新する。

目的:

- 壁面反射
- 狭い隙間からの回折
- 障害物裏の音の影
- 複数反射
- 材質別の吸音
- 声の周波数による波長差

完成版はWebGLまたはWebGPUのping-pong textureでGPU計算する。

### 9.3 採用判断

| 条件 | 採用方式 |
|---|---|
| 3時間ハッカソン | レイ・ウェーブハイブリッド |
| 発表後の改良 | WebGL FDTD |
| 低性能端末 | レイ方式 |
| 物理表現重視 | FDTD |
| デモ安定性重視 | 固定マップ + レイ方式 |

## 11. ステージ生成

### 11.1 生成対象

プレイ開始時に以下をランダム生成する。

- 部屋と通路の接続
- 棚や障害物の配置
- 忘れ物の位置
- 出入口
- 幽霊の初期位置
- 危険床
- 壁材質
- 安全地帯

### 11.2 MVP生成方式

完全な自動迷路生成ではなく、事前定義した部屋ブロックを組み合わせる。

例:

- 入口ブロック
- 長い商品棚ブロック
- 分岐ブロック
- 倉庫ブロック
- 園芸売り場ブロック
- レジブロック
- 忘れ物候補ブロック

毎回3〜6ブロックをシード値に基づいて接続する。

### 11.3 生成後検証

- 入口から忘れ物まで到達可能である。
- 忘れ物から入口まで帰還可能である。
- ロボットが通れる幅がある。
- 幽霊が通路を封鎖しても代替経路が最低1つある。
- 忘れ物が壁内へ配置されていない。
- 安全地帯が最低1か所ある。
- クリア不能なら再生成する。

### 11.4 シード

- 各マップにシード値を付与する。
- リザルト画面へシードを表示する。
- 同じシードを再入力して再挑戦できるようにするのは拡張要件とする。

## 12. 忘れ物システム

### 12.1 依頼

プレイ開始前に1件の依頼を表示する。

例:

- 銀色の工具箱
- 赤い手袋
- 子どものぬいぐるみ
- 古い写真立て
- 鍵束
- スマートフォン
- 指輪
- 顧客のノート

### 12.2 探知

忘れ物へ音波が当たると通常物体と異なる反応を返す。

- 二重の波紋
- 黄色の輪郭
- 特殊な効果音
- 短い周期の点滅
- 近距離でロボットのセンサーが反応

### 12.3 回収

- 忘れ物付近でEキーを押すと取得する。
- 取得後はロボットの速度を10〜25%低下させる。
- 幽霊の行動レベルを1段階上げる。
- 入口の方向を示す弱い反響を一定間隔で出す。
- 忘れ物を持って入口へ到達するとクリア。

## 13. 幽霊システム

### 13.1 共通AI状態

```text
待機
 → 徘徊
 → 音を検知
 → 音源位置を調査
 → ロボットを発見
 → 追跡
 → 攻撃
 → 見失う
 → 周辺捜索
 → 徘徊へ戻る
```

### 13.2 音の検知

幽霊は単純な円形範囲ではなく、音波計算で到達したエネルギーを利用して検知する。

```text
ghostAwareness += receivedSoundEnergy × sensitivity
```

- 壁の反対側では音が弱くなる。
- 金属棚の反射により、直接見えない場所でも気づく場合がある。
- 大声ほど広い範囲の幽霊が調査状態へ入る。
- 無音になっても、幽霊は最後の音源位置を一定時間調べる。

### 13.3 幽霊の種類

#### G-01 追跡者

- 音源位置へ高速接近する。
- ロボットを発見すると追跡する。
- 接触中は継続ダメージを与える。
- MVP必須。

#### G-02 道塞ぎ

- 一定時間、通路へ幽霊障壁を生成する。
- 障壁を通過できない。
- 強い音波を複数回当てると解除できる。
- Should Have。

#### G-03 呪い床

- 通過すると耐久値を減らす。
- 移動速度を低下させる。
- 音波を当てないと見えない。
- Should Have。

#### G-04 ポルターガイスト

- 棚や商品を動かして通路構造を変える。
- 偽物の反響を作る。
- 拡張要件。

#### G-05 音喰らい

- 一定範囲の波を吸収する。
- レーダーに不自然な空白を作る。
- 拡張要件。

### 13.4 攻撃

- 接触時に耐久値を減少させる。
- 連続接触では0.5秒ごとにダメージを与える。
- 被弾時は画面ノイズと操作遅延を短時間発生させる。
- 耐久値0でゲームオーバー。

## 14. ロボット要件

### 14.1 基本ステータス

| ステータス | 内容 |
|---|---|

- バッテリー: 100

- 非常灯消費: 8〜15 / 秒

## 15. 非常灯

### 15.1 機能

- ロボット周辺を直接照らす。
- 音波よりも正確に周囲を確認できる。
- 点灯中は近くの幽霊へロボット位置を通知する。
- バッテリーを急速に消費する。
- 緊急脱出用のハイリスク機能とする。

### 15.2 MVP扱い

3時間MVPではカット可能。  
時間に余裕がある場合のみShift長押し、またはLキーで実装する。

## 16. 勝敗条件

### 16.1 クリア

- 忘れ物を取得している。
- 入口へ到達する。
- ロボットの耐久値が1以上残っている。

### 16.2 ゲームオーバー

- 耐久値が0になる。
- 制限時間を超過する。
- バッテリーが0になり、帰還不能になる。
- 幽霊に一定時間拘束される。

### 16.3 制限時間

- MVP: 180秒
- Demo Mode: 90秒
- 拡張版: 難易度により3〜10分

## 21. AI利用要件

AIはコアゲーム進行の必須条件にしない。

| 用途 | 必須度 | 入力 | 出力 | 失敗時 |

| 声紋・感情推定 | 禁止 | 使用しない | 使用しない | - |

## 30. 受け入れ基準

### 30.1 起動

- `npm run build` が成功する。
- 環境変数なしでタイトルから結果まで到達できる。
- 1440×900でUIが重ならない。
- 390×844で横スクロールが発生しない。

### 30.2 マイク

- マイク許可後、音量メーターが反応する。
- 発声から200ms以内を目安にPingが開始する。
- 音声を保存または送信しない。
- マイク拒否時は手動モードへ自動切替する。

### 30.3 波

- 波がロボット中心から円形に広がる。
- 壁に当たった方向の波面が欠ける。
- 当たっていない部分は先へ進む。
- 反射波は元波より弱く表示される。
- 距離に応じて波が薄くなる。
- 開口部を抜けた波が簡易的に広がる。
- 波が当たった地形だけ一定時間表示される。

### 30.4 ゲーム

- 声を出していない状態ではロボットが停止する。
- 忘れ物を取得できる。
- 忘れ物を持って入口へ戻ると成功する。
- 幽霊が大きな音へ反応する。
- 幽霊接触で耐久値が減少する。
- 耐久値0または時間切れで失敗する。
- 再挑戦できる。

### 30.5 データ

- 結果がLocalStorageへ保存される。
- API未設定時に固定依頼と固定コメントを使う。
- Supabase未設定でもエラーにならない。

## 33. 最終設計方針

この作品で最優先するのは、物理学的な完全再現ではなく、プレイヤーが次の因果関係を直感的に理解できることである。

```text
声を出す
 → 波が広がる
 → 空間が見える
 → 幽霊にも聞こえる
 → 危険が近づく
```

ただし、波の欠け方、反射、減衰、開口部からの広がりは、可能な範囲で物理法則に基づいて表現する。

3時間MVPでは「物理近似型レイ・ウェーブハイブリッド」を採用し、完成後にFDTDへ置き換えられる構造を保つ。

本ゲームの中心体験は、次の葛藤である。

> **暗闇を知るために声を出すか。  
> 生き残るために声を止めるか。**
